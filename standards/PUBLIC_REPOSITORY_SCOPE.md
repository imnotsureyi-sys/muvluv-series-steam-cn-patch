# 公开仓库范围

公开 GitHub 仓库只放用户、测试者和贡献者真正需要的资料。内部交接、临时审计和完整游戏资源不放入公开仓库树。

## 公开保留

- 根目录 `README.md`：下载、项目说明和反馈入口。
- `patch-sources/`：公开维护的 JP-CN 补丁源表。
- `tools/`：可复用的提取、写回、差分、字体检查工具。
- `glossary/`：共通 JP-CN 术语表。
- `standards/`：翻译、源表和公开仓库范围规范。
- `chapters/`：章节工作流状态卡片和公开说明。

## 本地保留

- Codex 内部交接提示词。
- 本地分支和工作区操作流程。
- 提交模板和发布操作笔记。
- ATE / 参考补丁研究笔记。
- 一次性审计脚本和生成报告。
- 旧 AI 翻译批次和中间输出表。
- 完整游戏资源、RIO / egpack 重打包输出、缓存和测试安装目录。

## 发布资产

补丁 zip、`payload/`、安装脚本和完整补丁目录属于发布资产，应放在 GitHub Release 或网盘镜像中，不提交到仓库树。
