# 补丁源表规则

本文说明公开补丁源表的字段、修改和审计要求。源表是从 JP 原文定位到当前 CN 文本的工作基准，不是随意整理后的阅读表。

## 源表位置

TDA00-03 当前源表放在 `patch-sources/`。后续章节如进入公开维护阶段，也应放入同类目录或在章节 README 中说明位置。

每个源表应保留：

- 稳定行顺序。
- 游戏文本 ID 或 stable id。
- 来源包、RIO 文件、egpack 或其他资源容器。
- 场景定位信息。
- 日文说话人字段。
- 日文原文。
- 当前中文文本。

内部审计列、review 状态、临时标记和一次性处理字段不放在公开源表中。

## 修改规则

- 只修改目标章节的源表。
- 保持行数和稳定定位字段对齐。
- 保留控制符、ruby / 显示标记、换行标记和原文功能性符号。
- 不因 JP 文本看起来为空就删除台词，必须先确认实际显示槽。
- 不用英文槽、旧中文或模糊匹配写回中文。
- 修改前必须定位 chapter、csv_row、id / stable_id、resource_file、scene、speaker_jp、jp_text、cn_text。

## 修改后 QA

修改后至少检查：

- Text ID Not Found。
- 空 `cn_text`。
- 行顺序或定位错位。
- 异常重复文本。
- 乱码或编码损坏。
- 英文整句残留。
- 不该残留的日文假名。
- 控制符损坏。
- 原文符号被错误替换或丢失。
- 说话人、人名、军衔和术语一致性。

完整补丁包、payload、安装脚本和 repack 输出应通过 GitHub Release 或网盘镜像分发，不提交到仓库树。
