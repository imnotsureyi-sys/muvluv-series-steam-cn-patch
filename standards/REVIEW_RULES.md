# 独立审核规则

本文规定候选 CN 完成后的语义和中文质量审核。审核与初译是两个阶段；初译规则见 `TRANSLATION_RULES.md`。

## 1. 审核边界

1. 审核对象必须同时包含 JP 原文、候选 CN、scene、speaker 和上下文。
2. 审核者必须重新阅读 JP，不能只润色中文。
3. 英文槽和旧中文不能作为裁决依据。
4. 译者自检不等于独立审核。条件允许时，应由不同对话、不同人员或至少在独立轮次中重新判断。
5. 审核阶段不默认写回资源、打包或安装游戏。

## 2. 审核输入

每行至少需要：

- `chapter`
- `csv_row`
- `id / stable_id`
- `resource_file`
- `scene`
- `speaker_jp`
- `jp_text`
- 候选 `cn_text`
- 当前章节术语基线版本

缺少 JP、定位或必要上下文时，不得判定通过。

## 3. 逐句结论

每行必须给出以下一种结论：

- `keep`：候选 CN 可以保留。
- `revise`：给出修改后的 CN，并说明关键原因。
- `question`：依据不足，列明需要确认的内容。

不能批量默认 `keep`。

## 4. `keep` 条件

必须同时满足：

1. JP 原意准确，没有误译、漏译或擅自增译。
2. 中文自然，放进上下文不突兀。
3. 人物语气、称谓和关系正确。
4. 术语符合已冻结基线。
5. 原文中的功能性符号和表达作用没有无故丢失。
6. 英文、拉丁缩写、外语句和カタカナ处理符合项目决定。
7. 没有明显日式句壳、翻译腔或错误旧译名。

## 5. `revise` 条件

出现以下任一问题时应修改：

- 误译、漏译、增译或逻辑关系错误。
- 中文不通顺、拗口、过度书面或与原文文体不符。
- 角色语气、身份或称谓不对。
- 术语、专名或固定表达不统一。
- 普通口语被翻成不自然的文绉绉表达。
- 有命名感或设定意义的词被磨掉。
- 原文英文、拉丁缩写或关键符号被错误删除或替换。

修改必须回到 JP 和上下文，不做只凭个人偏好的同义改写。

## 6. `question` 条件

- JP 可多解，需要后文或更多场景确认。
- 术语、称谓或说话人无法确定。
- 可能涉及固定梗、设定词或官方写法。
- 外语句是否保留、翻译或加注尚未决定。
- scene 归属、人物关系或文本槽用途不明确。
- 技术显示结果可能影响语义判断。

## 7. 审核检查维度

1. 日文准确性：主宾关系、否定、反问、条件、因果、转折和时间关系。
2. 信息完整性：人名、称谓、设定、情绪和剧情功能。
3. 中文自然度：口语、旁白、军令、誓词等是否符合文体。
4. 角色一致性：同一角色在相同关系和状态下是否保持稳定语气。
5. 术语一致性：是否符合章节冻结基线和公共总表。
6. 符号功能：`「」`、`『』`、括号等是否被无故删除或改成不同功能的符号。
7. 外语处理：BETA、CP、ID 等缩写是否无故丢失，外语句处理是否有明确决定。

控制符、编码、写回位置和打包完整性由 `TECHNICAL_QA_RULES.md` 检查，不因为语义审核通过而自动通过。

## 8. 审核输出

每批至少保留：

1. 决策记录：每行 `keep / revise / question`。
2. 修改清单：定位字段、旧 CN、新 CN 和修改理由。
3. 问题清单：所有待确认内容及所需证据。
4. 审核摘要：范围、总行数和三类结论数量。
5. 交接起点：下一批从哪个 `csv_row`、`scene`、`id / stable_id` 继续。

这些内部审核字段不加入公开补丁源表。

## 9. 章节审核完成标准

1. 所有候选 CN 都经过逐句审核。
2. 每行都有可追踪的 `keep / revise / question` 结论。
3. 所有 `revise` 已给出新 CN，而不是只指出问题。
4. 所有 `question` 都有明确待确认事项。
5. 完成全章术语、称谓、角色语气和重复表达的一致性复查。
6. 语义审核完成后，才将定稿文本交给技术写回和 QA。
