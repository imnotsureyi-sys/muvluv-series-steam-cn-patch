# 通用初译规则

本文只规定从 JP 原文产出第一版 CN 候选译文的流程。术语确认、独立审核、资源写回和技术质检分别见：

- `TERMINOLOGY_WORKFLOW.md`
- `REVIEW_RULES.md`
- `TECHNICAL_QA_RULES.md`

## 1. 适用范围

适用于 TDA00-03、photonflowers、photonmelodies 以及后续章节。章节可以补充本章角色语气、专属术语和资源特点，但不能降低本文的硬规则。

初译阶段只产出候选中文，不等于审核通过，也不默认写回资源、生成补丁或安装游戏。

## 2. 硬规则

1. JP 原文是唯一翻译依据。
2. 不使用英文槽兜底。
3. 不使用旧中文兜底。
4. 不使用模糊匹配兜底。
5. 旧 CN 只能作为待审核候选，不能反向决定 JP 含义。
6. 拿不准的术语、称谓、说话人或剧情含义必须标记为 `question`，不能硬猜。
7. 只处理当前明确指定的章节，不跨章节直接修改字幕正文。
8. 初译阶段不写回 RIO、egpack、FPD 或其他游戏资源。

## 3. 章节开工条件

开始正文初译前，应完成：

1. 确认章节、route、scene 顺序和文本来源。
2. 按 `TERMINOLOGY_WORKFLOW.md` 扫描全章专有名词。
3. 对照 `glossary/muvluv_jp_cn_terms.csv`，确认本章术语基线。
4. 将未确认术语列入问题清单；高频或关键术语未定时，不批量推进相关文本。
5. 记录本章主要人物关系、称谓和语气基准。

这里的“确认术语”针对人名、组织、军衔、机体、武器、作战名、称号、固定梗等可复用表达，不要求用户逐个确认普通词句。

## 4. 每批开始前

每批按完整 scene 或自然剧情段推进，不随机抽行。翻译前先确认：

- `chapter`
- `csv_row`
- `id / stable_id`
- `resource_file`
- `scene`
- `speaker_jp`
- `jp_text`
- 当前术语基线版本

并简要理解：

1. 场景发生在哪里、谁在场、正在发生什么。
2. 本段如何承接前文、引出后文。
3. 说话人与被称呼者的关系和身份差。
4. 当前情绪、冲突和台词目的。
5. 本批是否出现新术语、特殊称谓或外语表达。

## 5. 日文理解

1. 逐句确认主语、宾语、否定、反问、条件、因果、转折和时间关系。
2. 日文省略主语时，可以按上下文补出必要的中文主语，但不能乱补剧情。
3. 一句 JP 有多种理解时，应结合 scene、人物关系、情绪和剧情目的判断，不能只看字面。
4. 长定语、被动句、倒装和重复语气可以重组为自然中文，但核心信息和逻辑不能丢。
5. 不能省略人名、称谓、设定、情绪转折或对剧情有作用的信息。

## 6. 中文表达

1. 中文应自然、通顺，像正式汉化文本。
2. 忠实原意，但不机械保留日文句壳。
3. 可以拆句、调整语序、补必要主语和调整语气词。
4. 不能乱编剧情、添加设定或改变人物关系。
5. 普通对白应口语自然，避免无依据地写成古风或过度书面语。
6. 誓词、军令、贵族式说话、宗教或军事语境可以保留相应文风。
7. 旁白应清楚顺滑，避免堆叠抽象名词。
8. 角色台词要符合人物身份和当时状态，不能让所有角色使用同一种中文腔调。
9. 不能为了顺口磨掉有命名感、反复出现或具有设定意义的表达。

## 7. 称谓、外语和专名

1. `さん`、`ちゃん`、`様`、`殿`、`君` 等不能机械翻译，必须按人物关系和场景判断。
2. “桑”“先生”“同学”“大人”“阁下”“少尉殿”等都不是默认答案。
3. 原文刻意使用敬称制造距离、身份或滑稽效果时，应保留这种功能。
4. JP 原文直接出现的 BETA、CP、ID、VIP、DMZ 等拉丁字母或缩写通常保留。
5. 原文中的完整外语句原则上先保留；是否翻译或加注，列为候选决定。
6. カタカナ不默认音译。人名、品牌和固有名优先固定译名；称号、武器、组织和作战名可按已确认术语选择意译、音译或保留英文。
7. 新专名不得在初译中自行定稿，应进入术语流程。

## 8. 符号和技术边界

1. `jp_text` 和定位字段不得修改。
2. 原文中的引号、括号、书名号和其他功能性符号不能无故丢失或改变含义。
3. 初译时可以记录目标文本所需的符号，但不能假设所有资源格式都要求逐字符复制 JP 控制符。
4. `\n`、`\w`、`\p`、`\f`、`<01>`、`<03>` 等内容由 `TECHNICAL_QA_RULES.md` 按资源格式处理。
5. 初译通过不代表控制符、编码和写回位置已经通过技术验证。

## 9. 初译输出

每批至少保留：

1. 候选译文：当前批次的完整 CN 候选。
2. 定位信息：`csv_row`、`id / stable_id`、`scene`、`speaker_jp`、`jp_text`。
3. 问题清单：拿不准的术语、称谓、剧情理解和特殊表达。
4. 新术语候选：交给 `TERMINOLOGY_WORKFLOW.md` 处理。
5. 交接摘要：本批范围、总行数和下一批起点。

初译状态只使用：

- `translated`：已给出候选译文，等待独立审核。
- `question`：需要确认后才能定稿。
- `blocked`：缺少 JP、上下文、定位信息或其他必要依据。

初译阶段不使用 `keep / revise`，这两个结论属于独立审核。

## 10. 译者自检

提交审核前，译者只做基础自检：

1. 是否准确理解 JP 原意。
2. 是否有漏译、擅自增译或逻辑错误。
3. 中文是否通顺，角色语气是否明显跑偏。
4. 是否遵守已冻结的术语和称谓。
5. 原文功能性符号是否有明显遗漏。
6. 是否把不确定内容如实列为 `question`。

译者自检不能代替 `REVIEW_RULES.md` 规定的独立审核。

## 11. 初译完成标准

一个批次完成初译时，应满足：

1. 每行都有 `translated`、`question` 或 `blocked` 状态。
2. 所有可翻译行都有基于 JP 的候选中文。
3. 所有疑点均进入问题清单，没有静默猜测。
4. 新术语已进入术语候选流程。
5. 批次边界和下一批起点可追踪。
6. 尚未把候选译文当成审核通过文本或直接写回游戏资源。
