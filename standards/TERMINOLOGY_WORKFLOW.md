# 章节术语工作流

本文规定每个章节开始前和翻译过程中的术语确认、冻结与同步方式。目标是在章节内保持一致，并把可复用的确认术语沉淀到 MAIN 总术语表。

## 1. 文件和角色

- 公共总表：`glossary/muvluv_jp_cn_terms.csv`
- 公共总表只保留 `jp,cn` 两列，不加入理由、状态、来源或审核列。
- 章节候选表、理由、出处和待确认状态放在章节工作区或本地内部记录中。
- MAIN 负责总表、跨章节协调和最终同步。
- 章节对话只处理本章术语，不直接修改其他章节正文。

## 2. 哪些内容属于术语

应优先扫描：

- 人名、姓氏、昵称和称号。
- 国家、组织、部队、机构和派系。
- 军衔、职务和固定称谓。
- 机体、兵器、装备、设施和技术名词。
- 作战名、计划名、地名和事件名。
- 有命名感的固定表达、反复短语和作品内梗。
- 重要カタカナ词、外语词和缩写。

普通动词、形容词和只出现一次的日常表达通常不进入总术语表。

## 3. 章节开工前的全章扫描

正文初译开始前，先扫描本章全部 JP 文本：

1. 提取候选术语及其出现次数和典型上下文。
2. 与公共总表逐项比对。
3. 分类为：
   - `confirmed`：总表已有且适用于本章。
   - `candidate`：总表没有，需要确认译法。
   - `contextual`：同一 JP 可能因语境采用不同中文，需要记录适用条件。
   - `question`：含义、指代或固定写法仍不确定。
4. 对高频、关键剧情和跨章节术语优先确认。
5. 形成“本章术语基线”，记录日期或版本，随后才开始批量初译。

章节开工时要确认的是专名和固定表达，不需要让用户审批每个普通词句。

## 4. 译法确认依据

译法判断顺序：

1. 用户已经明确确认的译法。
2. 公共总表中已确认且语境一致的译法。
3. 作品内 JP 上下文和官方日文设定。
4. 已有可靠中文惯用译名或经确认采用的中文资料。
5. 在语义明确的前提下，根据人名、专名、称号等类别决定音译、意译或保留英文。

英文槽、旧中文和模糊匹配不能作为确认依据。查到的网络资料也只能作为候选证据，不能越过 JP 原文和用户已经确认的项目译法。

## 5. 翻译中的术语处理

1. 每批开始前检查本批术语是否都在本章基线中。
2. 新出现的专名先加入章节候选记录，不能在不同批次随意采用不同译法。
3. 拿不准时标记 `question`，不得硬猜或静默替换。
4. 已冻结术语若需修改，必须记录旧译、新译、原因和影响范围。
5. 修改跨章节公共术语前，由 MAIN 检查 TDA00-03 和其他章节的影响。
6. 术语变更后应重新检查本章所有出现位置，不能只改当前一句。

## 6. 冻结和版本更新

“冻结”表示当前章节按这版术语推进，并不表示永远不能修改。

需要更新基线时：

1. 先确认新译法。
2. 记录受影响的 JP 词条和章节范围。
3. 更新章节术语基线版本。
4. 扫描本章所有旧译变体。
5. 若属于可复用专名，再交给 MAIN 同步公共总表。

不得在没有记录的情况下直接覆盖已确认术语。

## 7. 同步公共总表

满足以下条件的词条可以进入公共总表：

1. JP 和 CN 对应明确。
2. 译法已经确认。
3. 具有跨场景或跨章节复用价值。
4. 不依赖只在某一句成立的临时语境。

同步时只写 `jp,cn`。来源、理由、适用条件和讨论记录保留在内部候选表或交接记录，不污染公共两列表。

## 8. 章节术语完成标准

章节进入最终审核前，应满足：

1. 高频和关键专名都有明确译法或仍在可见的 `question` 清单中。
2. 同一术语没有无依据的多种 CN 变体。
3. 称谓差异可以由人物关系和场景解释。
4. 本章新增的可复用术语已提交 MAIN 确认。
5. 公共总表与章节最终采用译法没有未说明的冲突。
