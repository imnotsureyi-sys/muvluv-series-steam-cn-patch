# 项目规范

这里保存公开仓库中需要长期保留的通用规则。它们用于说明翻译、补丁源表、工具和公开仓库范围的最低要求，方便维护者、反馈处理者和后续章节工作对齐。

## 文件

- `TRANSLATION_RULES.md`：通用翻译、审校、分批推进和审计规则。
- `PATCH_SOURCE_RULES.md`：补丁源表的字段、修改和 QA 规则。
- `PUBLIC_REPOSITORY_SCOPE.md`：公开 GitHub 仓库应该保留和不应该保留的内容。

## 原则

- JP 原文是唯一翻译依据。
- 不使用英文槽、旧中文、模糊匹配作为兜底。
- 修改前必须能定位章节、CSV 行、文本 ID、资源文件、场景、说话人、JP 原文和当前 CN。
- 修改后必须审计错位、空文本、重复、乱码、控制符、英文/缩写、日文残留和术语一致性。
- 完整补丁包、payload、repack 输出和本地交接材料不放入公开仓库树。
