# EGPACK 多语言表格式

本文记录 TDA00-03 原始 EGPACK 和现有补丁中已经验证的结构。目标是让文本提取和写回可以按字段复现，而不是依赖可见字符启发式规则。

## 文件头

- 文件 magic：`EPK\0`
- 偏移 `8..11`：TDA00-03 标准文件中使用的小端 32 位文件总长度
- TDA00-03 原始文件和当前补丁中，声明长度等于实际字节数
- 表头列名：`de`、`en`、`es`、`fr`、`id`、`it`、`jp`、`pt`、`pt_br`、`zh_hans`、`zh_hant`

`id` 是文本记录 ID 列，不是印度尼西亚语槽。

## 字段键

字段键是列名 CRC32 的小端字节：

| 字段 | 文件中的十六进制键 |
|---|---|
| `de` | `8b29907d` |
| `en` | `42c159f3` |
| `es` | `9bad5f90` |
| `fr` | `cece75cc` |
| `id` | `506739bf` |
| `it` | `34778ea2` |
| `jp` | `eee0ce8e` |
| `pt` | `2cde8e39` |
| `pt_br` | `a1508d55` |
| `zh_hans` | `629d650e` |
| `zh_hant` | `c1080190` |

已验证字段标记结构为：

```text
87 <4 字节 CRC32 小端键> A6 <UTF-8 值> 00
```

`00` 是字段值结束符。空槽仍然拥有字段标记，只是值长度为零。

## TDA00-03 记录顺序

当前四章使用下面的固定字段顺序：

```text
zh_hans, pt, pt_br, de, jp, zh_hant, es, it, id, fr, en
```

每条记录必须恰好拥有 11 个字段，其中 `id` 用于定位，另外 10 个字段作为语言槽导出。解析器逐条验证顺序，不从相邻 ID 推测记录边界。

## 文本与控制符

字段文本是 NUL 结尾 UTF-8。常见字面控制符包括：

- `\p`：页面或消息结束控制。
- `\w`：等待或分段控制。
- `\f`：纯控制记录，在 TDA02 和 TDA03 中确实存在。
- `\n`、`\r`：手动换行控制。

控制符属于字段值的一部分。提取时必须原样导出用于审计，不能从相邻文本补齐。中文写回时不使用字面 `\n`、`\r` 或真实 CR/LF，由引擎按文本框宽度自动换行；写回器会直接拒绝这些值。`\p`、`\w`、`\f` 分别承担结束、等待或纯控制用途，不属于手动换行。空字符串与只包含 `\f` 的字符串不是同一种记录。

## 资源分类

| 分类 | 典型形式 | 用途 |
|---|---|---|
| `scene` | `game_t00000`、`tda03_t00000` | 正文、旁白和多人消息文本 |
| `ruby` | `game_t00000_ruby` | 振假名和特殊读音 |
| `speaker` | `__speakers__.egpack`、`game_s00000` | 说话人名称 |
| `staffroll` | `__staffroll__.egpack`、`game_staff00000`、`tda02_staff00000`、`staff90000` | 片尾职员表 |

结构有效但命名未知的 ID 标记为 `unknown`，不会被丢弃。

## 为什么不能按 ID 前后抓文字

字段的物理顺序使一个 ID 前后同时存在当前记录和相邻记录的语言字段。EN 为空时，搜索“ID 后第一段可见文字”可能抓到下一条 JP；JP 只有 `\f` 时，搜索“ID 前最后一段可见文字”也可能抓到上一条 EN。

字段级工具直接读取 CRC32 键和 NUL 边界，因此空槽、控制槽和相邻正文不会发生串行错位。

## 写回原则

1. 重新解析原文件并唯一定位 `(relative_path, id, slot)`。
2. 校验原槽等于 `expected_text`。
3. 拒绝中文替换值中的字面 `\n`、`\r` 和真实 CR/LF。
4. 只替换值字节，不重建整个表。
5. 多处修改按偏移逆序执行。
6. 更新偏移 `8..11` 的文件总长度。
7. 重解析输出并用验证器比较唯一授权结果。

CSV 中保存的偏移只用于审计，不能取代写回前的重新定位。

## 已知边界

- 本文只描述 TDA00-03 已验证布局。
- 主任保护协会 ATE V2 是可正常使用的参考补丁。对本地 157 个 EGPACK 的只读探测显示，其成品文件多数沿用相同语言键，但部分文件保留旧声明长度，并出现附加键、重复字段或重排字段；这是引擎可容忍、但当前严格 TDA 写回器尚未覆盖的变体。
- 工具会准确报告 ATE 变体与已支持布局的差异，不尝试模糊恢复，也不会把“未支持”表述为参考补丁错误。
- `uistring.epk` 是另一种加密资源，不属于本文的 EGPACK 表。
- FPD 是外层资源容器，不属于本文的内层多语言表。
